| Introdução:

O Processo Unificado Ágil ou do inglês Agile Unified Processs adota uma filosofia serial (sequência linear de atividades) para o que é amplo interativo para que o é particular. Este processo possui 4 atividades clássicas adotadas pelo Processo Unificado.

Sendo eles:

  • Início
  • Elaboração
  • Construção
  • Transição

Dentro de cada atividade citada, a equipe itera ou se repete para alcançar a maior agilidade possível para entregar os incrementos para os usuários finais. As entregas devem ser enviadas o mais rápido possível.

A AUP possui cerca de seis atividades, na qual a equipe sempre irá trabalhar por cima.

As atividades são:

  • Modelagem Modelos UML que representam o negócio são criados. É importante que esses modelos sejam organizados, eficientes e adequados para permanecer ágil.

  • Implementação Esses modelos são traduzidos e colocados no código fonte.

  • Teste Depois de terem implementado o código fonte, a equipe faz uma série de testes usando cada método que puderem afin de achar algum erro gerado no código, para que assim eles possam corrigir e ajustar o codigo de acordo com os requisitos.

  • Aplicação: É a entrega de um incremento do software para o usuário final para depois ter um feedback e possivelmente ter futuras mudanças com base nisso.

  • Gerenciamento de Configuração e Gerenciamento de projeto: É o controle do que está acontecendo na produção do software, tendo como base o gerenciamento de alterções e possíveis riscos que podem ser gerados no código, assim como o controle de qualquer artefato persistente que são produzidos pela equipe.

  • Gerenciamento de Ambiente: Vai coordenar toda a infraestrutura do ambiente, como por exemplo o hardware dos pcs, ferramentas que podem ser utilizadas e outras tecnologias de suporte disponível para a equipe re sponsável.

| Vantagens e desvantagens

Essa técnica possui muitas vantagens como redução de risco, identificação antecipada dos problemas, maior eficiência e mais segurança. E assim como qualquer outra técnica ela também possui suas desvantagens como a necessidade de ter um bom gerente ótimo para o gerenciamento da equipe.

Vantagens

  • Maior Eficiência
  • Identificação antecipada dos problemas
  • Redução de riscos

Desvantagens:

  • Exige uma equipe experiente e alinhada com os processos
  • Exigem um bom gerente de projetos
  • Complexo e trabalhoso por pequenos projetos

| Diferenças para o RUP

O AUP é um processo mais flexiblizado que maximiza a eficiência da comunicação e a entrega incremental de valor para o cliente. Enquanto o RUP por outro lado é mais focado na documentação detalhada do projeto e é bem mais estruturado com as suas fases de projeto.

A escolha do AUP ou RUP depende de como vai ser o projeto que vai ser trabalhado e as suas características, a escolha do processo vai variar dependendo do tipo de proejeto que vai ser focado.

| Impacto nas empresas

O AUP pode trazer muito lucro para as empresas devido a sua facilidade de adaptação nos diferentes tipos de sistemas que são solicitados pelos clientes. Alguma delas são:

  • Flexiblidade:Ele permite que mudanças repentinas como a mudança de um requisito solictado pelo cliente possam ser ajustados com o plano e as atividades conforme as mudanças nos requisitos do cliente ou do mercado.

  • Maior colaboração da equipe: A colaboração da equipe envolve todas as partes como os desenvolvedores, testadores, gerentes e até mesmo os clientes que é o ponto de começo do projeto. Dessse jeito ele permite uma comunicação mais amigável e melhora no desenvolvimento do sistema.

  • Maior adaptação a mudanças: Esse processo pode ter uma alta eficiaria para se adaptar em casa de mudança de requisitos, prioridades ou mudanças no mercado. Isso reduz o risco de projetos malsucedidos e aumentar a reputação da empresa.

Quais são as fases do ciclo do Processo Unificado Ágil?

O ciclo de desenvolvimento do Processo Unificado Ágil é composto por 4 fases principais: concepção, elaboração, construção e transição. Cada fase tem as suas próprias atividades, modelos, implementação, teste, implantação, gestão de configuração, gerenciamentos de projetos e ambiente.

Veja a seguir o que significa cada fase:

  • Concepção: Nessa fase é abordado os requisitos principais do projeto para se ter um escopo inicial. Como por exemplo a identificação de Stakeholders, análise de riscos e a elaboração de um plano de alto nível.

  • Elaboração: A fase de elaboração é onde os requisitos são aprofundados detalhadamente e o início do plano de projetos. A equipe analisa produndamente os requisitos em busca de um design para o sistema e a definição de quais requisitos vão ter alta prioridade. O objetivo é ter uma base sólida para o desenvolvimento.

  • Construção: A fase de construção é onde começa o desenvolvimento do sistema com entregas incrementais regulares de partes utilizáveis do sistema. Nessa fase também acontece a seção de testes na medida que o software é desenvolvido para que as entregas não estejam com defeitas e que sejam de qualidade desde do início..

  • Transição: Nessa fase é onde o software é testado de forma mais aprofundada, implementado e validada para o ambiente de produção. Analisam os potenciais riscos e vulnerabilidades do software antes que possa ser feito a entrega final para o cliente.
