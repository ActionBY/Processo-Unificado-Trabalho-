Introdução:

O Processo Unificado Ágil ou do inglês Agile Unified Processs adota uma filosofia serial (sequência linear de atividades) para o que é amplo interativo para que o é particular. Este processo possui 4 atividades clássicas adotadas pelo Processo Unificado.

Sendo eles:

  • Início
  • Elaboração
  • Construção
  • Transição

Dentro de cada atividade citada, a equipe itera ou se repete para alcançar a maior agilidade possível para entregar os incrementos para os usuários finais. As entregas devem ser enviadas o mais rápido possível.

A AUP possui cerca de seis atividades, na qual a equipe sempre irá trabalhar por cima.

As atividades são:

  • Modelagem Modelos UML que representam o negócio são criados. É importante que esses modelos sejam organizados, eficientes e adequados para permanecer ágil.

  • Implementação Esses modelos são traduzidos e colocados no código fonte.

  • Teste Depois de terem implementado o código fonte, a equipe faz uma série de testes usando cada método que puderem afin de achar algum erro gerado no código, para que assim eles possam corrigir e ajustar o codigo de acordo com os requisitos.

  • Aplicação: É a entrega de um incremento do software para o usuário final para depois ter um feedback e possivelmente ter futuras mudanças com base nisso.

  • Gerenciamento de Configuração e Gerenciamento de projeto: É o controle do que está acontecendo na produção do software, tendo como base o gerenciamento de alterções e possíveis riscos que podem ser gerados no código, assim como o controle de qualquer artefato persistente que são produzidos pela equipe.

  • Gerenciamento de Ambiente: Vai coordenar toda a infraestrutura do ambiente, como por exemplo o hardware dos pcs, ferramentas que podem ser utilizadas e outras tecnologias de suporte disponível para a equipe responsável.
